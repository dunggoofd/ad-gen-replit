  <action>
**Create middleware/clientScope.js:**

Async Express middleware. Import { getClientById, getDefaultClient } from ../database/clients.

Resolution order inside try block:
1. X-Client-Id header: if req.headers['x-client-id'] is present, call getClientById(parseInt(req.headers['x-client-id'], 10))
2. active_client_id cookie: if client is still null and req.cookies.active_client_id is present, call getClientById(parseInt(req.cookies.active_client_id, 10))
3. DB fallback: if client is still null, call getDefaultClient()

If client is null after all three steps: return res.status(503).json({ error: 'No client workspace available. Create a client first.' })

If client resolved: set req.clientId = client.id and req.client = client, then call next()

Catch block: call next(err) — Express 5 forwards to error handler automatically.

module.exports = { clientScope }

**Update server.js:**

Add one require line after the existing database/init require:
  const { clientScope } = require('./middleware/clientScope');

Do NOT mount clientScope globally. Do NOT remove or reorder any existing middleware. Do NOT add any route for /api/clients yet (that is Phase 3). The import is added now so Phase 3 plans can reference it.

Anti-patterns to avoid:
- Do NOT use app.use(clientScope) globally — this would break /api/health
- Do NOT parseInt without the NaN guard (already handled in getClientById)
- Cookie name must be exactly 'active_client_id' (not 'clientId' or 'client_id')
