  <name>Task 1: Create database/init.js with pg Pool and initDb()</name>

  <action>
Create the `database/` directory and `database/init.js`.

The module must:

1. Require `pg` and destructure `Pool`.
2. Create a single shared `pool` instance:
   ```javascript
   const pool = new Pool({
     connectionString: process.env.DATABASE_URL,
     ssl: process.env.DATABASE_URL ? { rejectUnauthorized: false } : false,
   });
   ```
   SSL note: Set `rejectUnauthorized: false` whenever DATABASE_URL is present — this handles both Replit managed Postgres and Neon.tech which both use self-signed or hosted certs. Do NOT use `NODE_ENV === 'production'` as the SSL gate — DATABASE_URL presence is the correct signal (local dev with no DATABASE_URL uses no SSL).

3. Attach an error listener immediately after pool creation:
   ```javascript
   pool.on('error', (err) => {
     console.error('[db] Unexpected pool error:', err.message);
   });
   ```
   This is mandatory — without it, an idle client error becomes an unhandled exception that crashes Node.

4. Define `async function initDb()` that:
   - Acquires a client: `const client = await pool.connect()`
   - Uses try/finally to always release: `client.release()` in `finally`
   - Inside try: runs `await client.query('SELECT NOW()')` as a connectivity probe
   - Logs `[db] Connected to PostgreSQL` after the probe succeeds
   - Runs the `clients` table DDL (first table; Phase 2 will add all remaining tables):
     ```sql
     CREATE TABLE IF NOT EXISTS clients (
       id SERIAL PRIMARY KEY,
       name TEXT NOT NULL,
       created_at TIMESTAMPTZ DEFAULT NOW()
     )
     ```
   - Logs `[db] Schema initialized` after DDL completes

5. Export: `module.exports = { pool, initDb };`

Critical patterns from research:
- `client.release()` MUST be in `finally` — if the query throws and release is outside finally, the client leaks and the pool exhausts over time
- Do NOT wrap pool creation in try/catch — pool constructor is synchronous and cannot fail; only `pool.connect()` can fail
- `SELECT NOW()` is the correct connectivity probe — it's cheap, read-only, and always available
